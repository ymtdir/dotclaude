# 要件定義書作成スキル

高品質な要件定義書を作成するためのスキルです。

## 前提条件

- **推奨**: `docs/ideas/*.md` にアイデアや要件メモが配置されている
- ファイルがない場合は対話形式で要件をヒアリングする

## 出力ルール

`../rules/document-generation-guide.md` を参照。

**対象ファイル**: `docs/requirements.md`

## 参照ファイル

- **テンプレート**: `./assets/template.md`
- **作成ガイド**: `./references/guide.md`

## 要件定義書作成プロセス

### 1. docs/ideas/*.md の確認

```
docs/ideas/ 内のマークダウンファイルを全て読む
```

### 2. 要件定義書ドラフトの生成

`docs/ideas/*.md`の内容をもとに、テンプレート（`./assets/template.md`）に従って要件定義書を生成します。
詳細な記述ポイントは `./references/guide.md` を参照してください。

### 3. 要件定義書のレビューと改善

生成された要件定義書を以下の観点でレビューします:

1. プロダクトビジョンは明確か
2. ターゲットユーザーは具体的か
3. 成功指標は測定可能か
4. 機能要件は実装可能なレベルまで詳細化されているか
5. 非機能要件は網羅されているか

### 4. レビュー後の改善

1. 指摘された問題を一つずつ確認
2. 具体化が必要な箇所を改善
3. 改善後、再度レビューを実施
4. 問題がなくなるまで繰り返す

**注意点**: AIのレビューを鵜呑みにせず、最終的な判断は必ず人間が行う

## 品質チェックリスト

### ビジョンと目標

- [ ] プロダクトビジョンが明確で測定可能か
- [ ] 提供する具体的な価値が定義されているか
- [ ] ターゲット市場が明確か

### ターゲットユーザー

- [ ] ペルソナが具体的に定義されているか
- [ ] 現在の課題と期待する解決策が明確か

### 成功指標

- [ ] SMART原則に従ったKPIが定義されているか
- [ ] 測定方法が明確か

### 機能要件

- [ ] すべての機能がユーザーストーリー形式で記述されているか
- [ ] 受け入れ条件が測定可能な形で定義されているか
- [ ] 優先度(P0/P1/P2)が明確に設定されているか

### 非機能要件

- [ ] パフォーマンス基準が具体的な数値で定義されているか
- [ ] 信頼性とセキュリティ要件が明確か

## トラブルシューティング

### docs/ideas/ にファイルがない

ユーザーに `docs/ideas/` へアイデアや要件メモの配置を依頼してください。ファイルがない場合は対話形式で要件をヒアリングします。

### 要件定義書の粒度が不適切

- 粗すぎる場合: `./references/guide.md` のセクション別ガイドを参照して詳細化
- 細かすぎる場合: 詳細設計書（detailed-design）に委譲すべき内容を分離
