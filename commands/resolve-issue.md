# Issueの解決

このコマンドは、GitHub Issue の内容を読み取り、ブランチ作成から解決まで自動で行います。

## 手順

### ステップ1: 引数バリデーション

引数から Issue 番号を取得する。取得できない場合は以下を返して終了:

```
エラー: Issue 番号が指定されていません。

使い方:
  /resolve-issue <番号>

例:
  /resolve-issue 42
```

### ステップ2: issue-resolution スキルの実行

1. **issue-resolution スキル**をロード
2. スキルに Issue 番号を渡して解決プロセスを実行（Issue 解析 → ブランチ作成 → 原子的コミット → 品質チェック）

既存ブランチ衝突時は skill の A/B/C 選択肢（再利用 / 別名で新規作成 / 削除して再作成）にユーザーが回答するまで進まない。

### ステップ3: 結果の報告

skill 実行後、以下のフォーマットで報告する。

**成功時（テスト全 pass）**:

```
✅ Issue #[番号]を解決しました！

ブランチ: issue-[番号]-[説明]  ([既存再利用 / 別名で新規作成 / 新規作成] のいずれか)
コミット数: [数]  (git rev-list --count main..HEAD で取得)
テスト結果: ✅

PR作成の準備ができました。
必要に応じて /create-pr を実行してください。
```

**テスト失敗時**:

```
⚠️ Issue #[番号]の解決中にテスト失敗を検出しました。

ブランチ: issue-[番号]-[説明]
コミット数: [数]
テスト結果: ❌ [test / lint / typecheck のどれが失敗したか] 失敗

失敗内容:
[失敗出力の要約]

修正方針を提示後、ユーザー承認を得てから続行してください。
```

**既存ブランチ衝突時（ユーザー選択待ち）**:

```
⏸ Issue #[番号]の解決を一時停止しました。

既存ブランチ `issue-[番号]-*` が検出されました。以下から選択してください:
- (A) 既存を再利用
- (B) 別名 `-v2` で新規作成
- (C) 既存削除して作り直し（承認必須）
```
