# ドキュメントレビュー

引数: ドキュメントパス（例: `/review-doc docs/requirements.md`）

## 手順

### ステップ1: 引数バリデーションと存在確認

1. 引数からドキュメントパスを取得する。引数が無い場合は以下を返して終了:

   ```
   引数が指定されていません。

   使い方:
     /review-doc <ドキュメントパス>

   例:
     /review-doc docs/requirements.md
   ```

2. 指定されたパスの存在を確認する。存在しない場合は doc-review skill のエラー応答フォーマットを返して終了（doc-reviewer は起動しない）:

   ```
   ⚠️ レビュー対象 `<パス>` が見つかりません。

   確認事項:
   - パス指定が正しいか
   - ファイルがまだ未作成か

   正しいパスを指定して再実行してください。
   ```

### ステップ2: doc-reviewer サブエージェント起動

Agent tool を使用して doc-reviewer を起動する:

- subagent_type: `doc-reviewer`
- prompt: `<ドキュメントパス> をレビューしてください。`（doc-reviewer の入力契約に合わせる）

### ステップ3: レビュー結果の要約

doc-reviewer の返却（出力契約）を以下のフォーマットで要約してユーザーに報告する:

```
✅ <ドキュメントパス> のレビューが完了しました。

【総合スコア】: <N>/5
【評価サマリ】:
  - 完全性 / 明確性 / 一貫性 / 実装可能性 / 測定可能性 の ✅/⚠️/❌
【必須対応】: <件数> 件

詳細は doc-reviewer の返却レポートを参照してください。
```
